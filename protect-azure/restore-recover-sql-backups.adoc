= 还原和恢复SQL Server数据库
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
您可以使用SnapCenter还原 SQL Server 数据库。数据库还原是一个多阶段过程，它将所有数据和日志页从指定的 SQL Server 备份复制到指定的数据库。

.开始之前
* 使用属于 SMB AD 活动目录域且具有设置文件权限权限的活动目录用户配置目标实例。
* 在SnapCenter实例级别配置凭据。
* 如果 SQL 数据库位于 CIFS 共享上，则插件主机的 RunAS 帐户和 SQL 服务帐户（域帐户）需要是以下组件的一部分： `BUILTIN\Administrators` ONTAP中的 CIFS 服务器组。
* SMB 配置不支持目标实例的 SQL 身份验证。在SnapCenter中配置目标实例，使用具有所需权限的活动目录用户。
* 如果SnapCenter插件服务帐户不是活动目录用户，请确保在恢复到备用主机时，具有对源卷的完全控制权限的用户可用。


.步骤
. 在左侧导航窗格中，单击 * 资源 * ，然后从列表中选择相应的插件。
. 在“资源”页中，从“视图”列表中选择*Database或*Resource Group*。
. 从列表中选择数据库或资源组。
. 从管理副本视图中，选择*从存储系统备份*。
. 从表中选择备份，然后单击 image:../media/restore_icon.gif["还原图标"] 图标。
. 在还原范围页面中，选择以下选项之一：
+
.. 选择“将数据库还原到创建备份的同一主机”可将数据库还原到同一 SQL 服务器。
.. 如果要将数据库还原到同一主机或其他主机上的另一个 SQL 服务器（该主机用于备份），请选择“将数据库还原到备用主机”。


. 在恢复范围页面中，选择以下选项之一：
+
.. 如果只需要还原完整备份而不需要任何日志，请选择 * 无 * 。
.. 选择*所有日志备份*最新备份还原操作、在完整备份后还原所有可用的日志备份。
.. 选择 * 通过日志备份 * 以执行时间点还原操作，此操作将根据备份日志还原数据库，直到具有选定日期的备份日志为止。
.. 选择 * 按特定日期至 * 以指定事务日志不应用于已还原数据库的日期和时间。
.. 如果选择了 * 所有日志备份 * ， * 通过日志备份 * 或 * 在特定日期之前 * ，并且日志位于自定义位置，请选择 * 使用自定义日志目录 * ，然后指定日志位置。


. 在操作前和操作后页面中、指定所需的详细信息。
. 在通知页面的 * 电子邮件首选项 * 下拉列表中，选择要发送电子邮件的场景。
. 查看摘要，然后单击 * 完成 * 。
. 使用 * 监控 * > * 作业 * 页面监控还原过程。

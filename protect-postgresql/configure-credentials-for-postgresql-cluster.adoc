= 为 PostgreSQL 集群配置凭据
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
必须为 PostgreSQL 集群配置集群用户凭据才能执行数据保护操作。

.开始之前
* 如果 PostgreSQL 集群未配置有效凭据，则仅为自动发现的资源显示红色挂锁图标。如果在后续发现操作中，发现配置的凭据不正确或未提供对集群本身的访问，则会重新出现红色挂锁图标。
* 您必须配置有效的 PostgreSQL 群集凭据才能保护资源或将其添加到资源组以执行数据保护操作。
* 凭据中配置的 PostgreSQL 用户必须具有足够的权限才能访问群集并执行备份和还原操作。用户至少应具有 *SUPERUSER* 角色或同等权限（例如 *pg_read_all_data*、*pg_write_all_data* 和 *replication* 权限）。
* SnapCenter 使用配置的凭据通过 *psql* 或 *libpq* 客户端连接到 PostgreSQL 实例。请确保在插件主机上安装了 PostgreSQL 客户端工具。



NOTE: 在配置集群之前，您必须首先将 PostgreSQL 凭据添加到 SnapCenter。

.步骤
. 在左侧导航窗格中，选择 *Resources*，然后从列表中选择 SnapCenter Plug-in for PostgreSQL cluster。
. 在资源页面中，从 * 视图 * 列表中选择资源类型。
. (可选) 选择 image:../media/filter_icon.gif["筛选器图标"] 并选择主机名。
+
然后，您可以选择 image:../media/filter_icon.gif["筛选器图标"] 来关闭过滤器窗格。

. 选择集群，然后选择 *Configure credentials*。
. 选择已在 SnapCenter 中为 PostgreSQL 集群用户配置的凭据。凭据必须包含具有执行数据保护操作所需权限的有效用户名和密码。
. 选择 * 确定 * 。


您可以从拓扑页面修改集群配置。
